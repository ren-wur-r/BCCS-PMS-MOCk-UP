# 角色權限定義（填寫版）

> 請補齊每個角色的「可見主模組」與「可用操作」。
> 模組與操作可用逗號或條列，保持一致即可。
> 每個角色都可見 Dashboard，可以看見的東西參考`/Users/r_r/Downloads/ProjectManagerSystem-prod/menuAndDashboard.md`

## Admin

- 可見主模組：全部
- 可用操作：全部

## 總經理

- 可見主模組：Dashboard
- 可用操作：主要是從 Dashborad 那邊看到已經分類好的狀態，比如商機統計（分百分比）、公司全專案（按風險等級），上述的資訊頁可以按照區域、人員去做分析，然後如果點擊進去則會顯示詳情。

## 各處處長

- 可見主模組：自己處的人員彙整，比如行銷處長的 Dashboard 就可以看到電銷打了幾筆轉給業務，然後協銷部產品經理陪業務出去幾次，正式成案百分比、活動辦了幾場，活動轉化率多少（轉給電銷）又派給業務
- 可用操作：可以新增專案、可以指派專案任務給自己底下員工，如果是其他部門則只能指派部門，再讓各部門主管去分派

## 各部門經理

- 可見主模組：專案管理、個人 Dashboard、系統設定裡面的專案標準階段
- 可用操作：只能針對自己部門成員做指派、也可以 CRUD 自己被指派到的專案，可以 CRUD 專案標準階段與產出。

## 活動人員

- 可見主模組：活動管理、個人 Dashboard
- 可用操作：CRUD

## 電銷人員

- 可見主模組：電銷管理、個人 Dashboard
- 可用操作：CRUD

## 顧問

- 可見主模組：專案管理、個人 Dashborad
- 可用操作：CRUD

## 產品經理

- 可見主模組：專案管理、產品管理(還沒新增)、個人 Dashboard
- 可用操作：CRUD

## 工程師

- 可見主模組：專案管理、個人 Dashboard
- 可用操作：CRUD

## 資深顧問

- 可見主模組：專案管理、個人 Dashboard
- 可用操作：CRUD
