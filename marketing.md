# 行銷管理模組（前端測試版）說明

此文件提供行銷管理模組於前端測試階段的版面與互動規格。後端資料由營運單位透過現有的 EDM/Teams 匯入流程填入，因此本說明聚焦在畫面結構、設計風格與假資料需求，讓前端能獨立交付一版可操作的樣貌供評估。

---

## 設計風格

- **色彩**：延續 PMS 既有 Tailwind 配色，背景使用 `#F5F7FB`，內容卡片為白底搭配柔和陰影 (`shadow`, `drop-shadow-sm`)。重點數字搭配品牌藍 `#0EA5E9`、成功綠 `#22C55E`，商機百分比使用藍/靛/琥珀/翠綠建立 10%/30%/70%/100% 的層次。
- **排版**：頁面四周維持 24px padding，卡片內距 24px，查詢區塊置於頂部並以 `gap-4` 排列。表格採 `table-fixed table-sticky`，頂部欄位固定，內容區可滾動。
- **元件語言**：日期選擇器統一採 `@vuepic/vue-datepicker`（range 模式，32px 高度），下拉使用 `select-box`，文字輸入為 `input-box`。列表 hover 以 `hover:bg-cyan-50` 提示可點擊，動作按鈕統一使用 `btn-search / btn-cancel / btn-add` 等既有樣式。
- **頁籤**：所有活動/名單詳細頁皆以 `ActivityDetailTabs` 呈現「檢視活動」、「檢視名單」兩個 tab，保持一致體驗並取代過去的「檢視」按鈕。

## 功能結構

### 1. 活動管理（/crm/activity/activity）

| 畫面 | 核心互動 | 假資料欄位 |
| --- | --- | --- |
| 活動列表 | 日期區間、名稱、類型篩選；整列點擊導向詳情；分頁與每頁筆數切換 | 活動 ID、名稱、類型、開始/結束時間、期望名單、實際名單、轉電數、商機數 |
| 詳情 Tab：檢視活動 | 顯示活動資訊（含地點、內容、產品清單）；未開始前允許切換編輯模式 | 活動基本欄位 + 活動產品列表（名稱、主/子分類、成本/售價） |
| 詳情 Tab：檢視名單 | 以 `CrmActivityPipelineList` 呈現名單表格，支援多選、轉電、刪除、分頁 | 名單狀態、客戶/窗口資訊、電話、Email、部門、職稱、名單 ID |
| 匯入 | EDM Excel、Teams 名單兩種匯入管道，結果使用 `ImportEdmResultModal`、`ImportTeamsResultModal` 呈現成功/失敗筆數 | 匯入欄位需符合現有後端格式（姓名、Email、電話、公司、職稱等） |

### 2. 電銷管理（/crm/phone/activity、/crm/phone/pipeline）

| 畫面 | 核心互動 | 假資料欄位 |
| --- | --- | --- |
| 活動列表 | 與活動管理一致，但統計欄位改為實際名單、轉電、撥打、商機 | 活動 KPI（Registered/Transferred/Phone/Opportunity） |
| 詳情 Tab：檢視活動 | 顯示 KPI 與活動資訊，供主管快速檢視 | 同上 |
| 詳情 Tab：檢視名單 | `CrmPhonePipelineList`，列可多選進行指派、轉派、匯入、刪除；整列點擊開啟名單詳情 | 名單狀態 Enum、客戶/窗口/電話/Email/部門、名單 ID |
| 名單詳情 | `CrmPhonePipelineDetail`，Tab 與活動列表共用，內容含客戶資訊、窗口資訊、指派紀錄、開發紀錄、產品、附件等 | 各區塊所需欄位皆要有假資料以驗證 accordion/表格樣式 |

### 3. 商機百分比條件

- `CrmPipelinePipelineDetail` 內的階段條件已調整為：  
  - **10%＝有需求**：確認需求即可。  
  - **30%＝有需求、有時間、有預算、已簡報**。  
  - **70%＝已簡報、已報價、議價中**。  
- 對應欄位（需求、時程、預算、簡報、報價、議價）使用下拉（已確認/未確認）並提供備註欄，前端需在欄位未符合時阻擋升級、顯示提示。
- 任何商機狀態更新都需同步刷新儀表板（GM/Sales）資料來源，確保 Dashboard 中的商機百分比與明細表會反映尚未轉專案的商機。

## 匯入與假資料策略

- **EDM 匯入**：可準備 JSON 陣列模擬 Excel 解析後的結果，欄位包括 `Name`, `Email`, `Phone`, `CompanyName`, `Department`, `JobTitle`, `SourceActivityID` 等，並在 Modal 中顯示成功/失敗筆數。  
- **Teams 匯入**：欄位包含 `TeamsUserId`, `DisplayName`, `Email`, `Company`, `Department`。  
- **名單狀態**：沿用 `DbAtomPipelineStatusEnum`，避免與後端枚舉不一致。  
- **Dashboard 模擬**：若無後端 API，可在 `mockDataStore` 或各頁面內定義靜態資料，提供 GM 與 Sales 儀表板測試用的 `PipelineStageStatistics`、`PipelineStageDetails`、`PersonalPipelineStageStatistics` 等欄位。

---

## 權限與導覽假設

1. 行銷管理選單標題顯示「行銷管理」，下含「活動管理」、「電銷管理」。  
2. 擁有 `CrmActivity` 或 `CrmPhone` 之 `Everyone` 權限者可瀏覽列表；`create`/`edit` 控制新增、批次處理按鈕是否顯示。  
3. Detail Tab 路徑統一為 `/crm/{activity|phone}/activity/detail/:id`，Tab 透過 query 或 router 設定切換，確保深層連結仍可使用。  
4. 無資料時以表格內 `td colspan` 顯示「無資料」，避免空白畫面。  
5. 所有異動需同步記錄於 `changelog.md` 以利營運追蹤。

---

## 測試建議

- 每個列表至少準備 10 筆活動、20 筆名單，便於驗證分頁與 hover 樣式。  
- 名單詳情需具備完整的指派紀錄、開發紀錄、產品紀錄，確保 QA 能檢查各 section 的間距與互動。  
- 先專注於亮色主題，若後續需要暗色模式可在此基礎延伸。  
- 若開發過程有任何 UI/流程調整，務必同步更新本文件與 `changelog.md`。
